* Agent指令撰写守则

- 多发小作文，减少聊天频率。
  - 做法：把自己的需求先写成文档。写的尽量清楚，考虑清楚所有的功能。写的越清晰越好。让AI直接读文档
  - 不要在一些琐碎细小的事情上浪费request次数，订阅都是很贵的，一个月只有不到20000次。
- 不再在乎代码的价值，但是更加在意prompt的可复用性。
  - 撰写通用的claude.md模板作为通用规范。
  - 比如，讲清楚：
    - 代码怎么写？
    - Git使用与项目管理？
    - test-driven？
    - 注释的规范性？
    - 。。。。
- 写prompt没必要要用英语。语言不影响性能。
- 提前规定好哪些终端命令可以用，哪些不能用。由此减少后续过程中存在的permission询问打断。



* Agent开发流程


1. 撰写任务描述文档，讲清楚自己所需要的目标都是什么。
2. 请AI基于任务描述文档进行细化，并二次生成一个新的任务描述文件，里面更详细地拆解了完成这些目标所需要的所有步骤。自我检阅这个文件。
3. 根据自己所定义的规范，让agent稳定地一步步将二次任务描述文件里的所有内容执行一遍。
