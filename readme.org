* Agent指令撰写守则

- 多发小作文，减少聊天频率。
  - 做法：把自己的需求先写成文档。写的尽量清楚，考虑清楚所有的功能。写的越清晰越好。让AI直接读文档
  - 不要在一些琐碎细小的事情上浪费request次数，订阅都是很贵的，一个月只有不到20000次。
- 不再在乎代码的价值，但是更加在意prompt的可复用性。
  - 撰写通用的claude.md模板作为通用规范。
  - 比如，讲清楚：
    - 代码怎么写？
    - Git使用与项目管理？
    - test-driven？
    - 注释的规范性？
    - 。。。。
- 写prompt没必要要用英语。语言不影响性能。
